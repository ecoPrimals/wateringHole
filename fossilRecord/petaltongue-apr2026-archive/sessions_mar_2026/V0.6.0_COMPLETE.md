## 📈 Metrics Explanation

### What Each Metric Means

**Motor Commands (Render)**:
- How many frames we've sent to display
- Increments every frame (~60/sec)
- Green ✅ after first frame

**Sensory Events (Sense)**:
- How many input events we've received
- Mouse moves, clicks, keyboard presses
- Green ✅ after first interaction

**Confirmation Rate (Loop %)**:
- What % of output is verified by input
- Formula: `(sensory_events / motor_commands) * 100`
- >50% = good, >90% = excellent

**User Visibility**:
- **Confirmed**: >90% confidence user sees us
- **Probable**: >50% confidence
- **Uncertain**: >0% confidence
- **Unknown**: No evidence

**Health Percentage**:
- Weighted score of all functions
- 100% = perfect operation
- >80% = excellent
- 50-80% = good
- <50% = degraded

### Why These Matter

**Sovereignty**: We don't assume - we measure  
**Transparency**: User sees our actual state  
**Adaptability**: Can respond to failures  
**Debugging**: Instant diagnosis of issues  
**Trust**: Honest about capabilities
