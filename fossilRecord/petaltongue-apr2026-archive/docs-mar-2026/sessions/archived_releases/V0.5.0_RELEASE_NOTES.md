## 🎊 Overview

petalTongue v0.5.0 marks a major architectural evolution, introducing **port-free inter-primal communication** via Unix domain sockets and complete **biomeOS integration**. This release eliminates network port dependencies for local primals while maintaining full backward compatibility.

---

## 🚀 What's New

### Port-Free Architecture
- **Unix Socket JSON-RPC Server**: Complete implementation of JSON-RPC 2.0 over Unix domain sockets
- **Zero Network Ports**: Local primals communicate via file system, eliminating port conflicts
- **File-System Discovery**: Infrastructure for discovering primals via `.sock` files
- **Graceful Fallback**: Maintains HTTP support for remote primals

### biomeOS Integration
- **Full Format Compatibility**: Native support for biomeOS topology JSON format
- **Extended Type System**: New `PrimalEndpoints`, `PrimalMetadata`, and `ConnectionMetrics` types
- **Smart Migration**: Automatic field migration for backward/forward compatibility
- **Mock Server**: Development REST API server for E2E testing

### APIs Implemented (4 Total)
1. `get_capabilities` - Query primal capabilities and version info
2. `get_health` - Check primal health status
3. `render_graph` - Render topology graphs (SVG/PNG/Terminal)
4. `get_topology` - Retrieve current topology data

## 🏗️ Architecture Changes

### New Crate: `petal-tongue-ipc`
Dedicated crate for inter-primal communication:
- `json_rpc.rs` - JSON-RPC 2.0 protocol implementation
- `unix_socket_server.rs` - Unix domain socket server
- Full async/await support with tokio
- Line-based protocol (newline-delimited JSON)

### Type System Extensions
```rust
// New in v0.5.0
pub struct PrimalEndpoints {
    pub unix_socket: Option<String>,
    pub http: Option<String>,
}

pub struct PrimalMetadata {
    pub version: Option<String>,
    pub family_id: Option<String>,
    pub node_id: Option<String>,
}

pub struct ConnectionMetrics {
    pub request_count: u64,
    pub avg_latency_ms: f64,
}
```

### Discovery Priority
Updated discovery coordinator with intelligent fallback:
1. **Unix Sockets** (preferred - port-free)
2. **mDNS** (automatic local network)
3. **HTTP** (environment hints)
4. **Mock** (development mode)

## 🚀 Upgrade Guide

### For Developers
```bash
# Pull latest code
git pull origin main

# Build with all features
cargo build --workspace

# Run tests
cargo test --workspace

# Optional: Test Unix socket server
cargo run --package petal-tongue-ipc --example unix_socket_demo
```

### For Operators
```bash
# Start petalTongue with Unix socket support
PETALTONGUE_UNIX_SOCKET=/tmp/petaltongue-node-alpha.sock \
cargo run --bin petal-tongue

# Test with mock biomeOS
cd sandbox/mock-biomeos && cargo run
# Server runs on http://localhost:3000
```

### Environment Variables
```bash
# Unix socket path (optional)
PETALTONGUE_UNIX_SOCKET=/tmp/petaltongue-{node_id}.sock

# biomeOS endpoint (optional)
BIOMEOS_URL=http://localhost:3000

# Discovery ports (optional)
PETALTONGUE_DISCOVERY_PORTS=8080,8081,8082

# Enable/disable mDNS
PETALTONGUE_ENABLE_MDNS=true
```

## 🎯 Use Cases

### 1. Port-Free Local Development
```bash
# Start petalTongue with Unix socket
PETALTONGUE_UNIX_SOCKET=/tmp/petaltongue.sock cargo run

# Other primals connect via file system
# No port conflicts, no firewall issues
```

### 2. biomeOS Integration
```bash
# Point to biomeOS instance
BIOMEOS_URL=http://localhost:3000 cargo run --bin petal-tongue

# petalTongue consumes and visualizes biomeOS topology
```

### 3. Multi-Modal Visualization
```bash
# Render topology via JSON-RPC
echo '{"jsonrpc":"2.0","method":"render_graph","params":{"format":"svg"},"id":1}' | \
  nc -U /tmp/petaltongue.sock
```
