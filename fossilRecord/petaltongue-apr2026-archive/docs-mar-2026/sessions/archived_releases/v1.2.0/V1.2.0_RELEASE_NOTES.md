## 🚨 Critical Fix

### Deadlock in StatusReporter

**Symptom**: GUI window created but never rendered frames  
**Root Cause**: Re-entrant mutex lock in `update_modality()`  
**Fix**: One scoped block to drop lock before nested calls  
**Impact**: Complete hang eliminated  

**Files Changed**:
- `crates/petal-tongue-ui/src/status_reporter.rs` (3 lines)

## ✨ New Features

### 1. Hang Detection System

**Purpose**: Detect when the application stops rendering

- Monitors time since last frame
- Threshold: 5 seconds without frame = hanging
- Logs diagnostic events automatically
- Displays prominent UI warning

**Usage**: Automatic - no configuration needed

### 2. FPS Monitoring

**Purpose**: Real-time performance visibility

- Tracks last 60 frames
- Calculates frames per second
- Color-coded display:
  - 🟢 Green: >30 FPS (healthy)
  - 🟡 Yellow: 15-30 FPS (degraded)
  - 🔴 Red: <15 FPS (poor)

**Display**: System dashboard shows live FPS

### 3. Diagnostic Event Log

**Purpose**: Debugging and post-mortem analysis

- Ring buffer of last 100 events
- Event types:
  - `hang_detected`
  - `hang_recovery`
  - (Extensible for future events)
- Timestamps and context included

**API**: `proprioception.get_diagnostic_events(count)`

### 4. Enhanced System Dashboard

**New displays**:
```
🧠 SAME DAVE Proprioception
Health: 85% ████████▌░
Confidence: 72% ███████▏░░

🎬 48.3 FPS (1,234 frames)  ← NEW!
✅ Motor | ✅ Sensory | ✅ Loop

⚠️  HANG WARNING (if detected)  ← NEW!
```

### 5. Conditional Diagnostics

**Purpose**: Verbose logging when needed, quiet in production

**Enable**:
```bash
PETALTONGUE_DIAG=1 ./target/release/petal-tongue
```

**Disable** (default):
```bash
./target/release/petal-tongue
```
