## 📝 Creating New Documentation

### File Naming Convention
- **Status docs**: `CURRENT_STATUS.md`, `KNOWN_ISSUES.md`
- **Session docs**: `*_NOV_3_2025.md` (with date)
- **Guides**: `*_GUIDE.md`
- **Plans**: `*_PLAN.md`
- **References**: `*_REFERENCE.md`

### Required Sections
1. **Header**: Title, date, status
2. **Quick Summary**: TL;DR in 30 seconds
3. **Main Content**: Organized with clear sections
4. **Navigation**: Links to related docs
5. **Bottom Line**: Key takeaways

### Where to Place
- **Root**: Essential, current documents
- **docs/**: Detailed technical documentation
- **specs/**: Specifications and design docs
- **archive/docs/**: Superseded or historical documents
